# 2026-04-19

## Changed
- 修复 BEditor 源码模式 CodeMirror 高度样式，确保内容高度交给外层滚动容器计算。
- 修复 BEditor 源码模式目录锚点无法跳转的问题，通过 CodeMirror 文档位置计算驱动外层滚动容器。
- 修复 BEditor 源码模式点击目录时同时触发 DOM 锚点和 CodeMirror 锚点滚动导致位置偏移的问题。
- 修复 BEditor 源码模式长文档后段目录跳转依赖 CodeMirror 估算高度导致滚动位置偏移的问题。
- 修复 BEditor 源码模式将 fenced code block 内伪标题误计入锚点编号，导致左侧目录跳转错位的问题。
- 修复 BEditor 源码模式滚动时目录高亮依赖 CodeMirror 可见行 DOM 的问题。
- 调整 BEditor 源码模式 Markdown 高亮主题色，匹配 Claude Light/Dark 配色。
- 为 BEditor 源码模式新增 fenced code block 语法高亮，复用 lowlight 语言词法结果映射到 CodeMirror decorations。
- 升级 `package.json` 中一批同 major 的前端与编辑器依赖版本，包含 AI SDK、Tiptap、Vite、Vue、UnoCSS 等，以降低升级风险并保持依赖新鲜度。
- 在保留现有 ESLint 与 Stylelint 版本不变的前提下，继续升级 Electron、TypeScript、Node 类型、`better-sqlite3`、`marked`、`vue-tsc` 等其余依赖。
- 兼容 TypeScript 6 对 `baseUrl` 的弃用校验，在根 `tsconfig` 中补充 `ignoreDeprecations` 配置以保持现有路径别名可继续编译。
- 将 Vue scoped 样式中的深度选择器改为 `:global(...)` 写法，避免 Lightning CSS 构建阶段保留 `deep` 语法并触发告警。
- 优化编辑器打包体积：将 Mermaid 预览改为按需加载，并将富文本编辑器与源码编辑器子面板拆为异步组件，降低编辑器入口 chunk 体积。
- 将 Vite 8 构建分包配置从已弃用的 `build.rollupOptions.output.manualChunks` 迁移到 `build.rolldownOptions.output.codeSplitting.groups`，与当前构建器能力保持一致。
- 修复开发环境启动地址对 `localhost` 解析的依赖，统一改为 `127.0.0.1`，避免当前环境下 Vite 与 Electron 开发启动失败。
- 补充 `electron/tsconfig.json` 的 `ignoreDeprecations` 配置，消除 Electron 主进程 watch 编译在 TypeScript 6 下的 `baseUrl` 弃用报错。
